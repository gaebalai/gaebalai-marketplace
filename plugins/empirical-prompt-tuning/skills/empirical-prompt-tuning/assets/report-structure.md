# Report Structure (보고 구조)

서브에이전트/외부 세션이 반환해야 하는 보고서의 정형 구조. dispatch 프롬프트에 강제로 포함시켜 응답 형태를 통제한다.

## 표준 보고 구조

```markdown
### 산출물
<생성물 본체 또는 실행 결과 요약 - 100~300자>

### 요건 달성
1. [critical] <항목명>: ○ / × / 부분적 (이유)
2. [critical] <항목명>: ○ / × / 부분적 (이유)
3. <항목명>: ○ / × / 부분적 (이유)
4. <항목명>: ○ / × / 부분적 (이유)
5. <항목명>: ○ / × / 부분적 (이유)

종합 정확도: <달성한 항목 수> / <전체 항목 수> = <%>
critical 항목 모두 ○ 여부: Yes / No

### 불명확점
- <막힌 부분 또는 해석 망설임 1>
- <막힌 부분 또는 해석 망설임 2>
- <막힌 부분 또는 해석 망설임 3>

(불명확점이 없으면 "없음" 명시)

### 재량 보완
- <항목 1>: 채운 값 = <값>, 근거 = <근거>
- <항목 2>: 채운 값 = <값>, 근거 = <근거>

(재량 보완이 없으면 "없음" 명시)

### 재시도
<횟수>회.
이유: <같은 판단을 다시 한 이유, 없으면 "재시도 없음">

### 메타 (Claude Code Task tool 사용 시 자동)
<usage>
total_tokens: <N>
tool_uses: <M>
duration_ms: <D>
</usage>
```

## 왜 이 구조가 필요한가

### 산출물

산출물 자체보다 **"산출물이 만들어졌는가"**의 확인용. 본문이 너무 모호하면 서브에이전트가 산출물을 만들지 못하고 질문만 늘어놓는다. 그것도 평가 신호.

### 요건 달성 (○/×/부분적)

`부분적`이 가장 중요한 신호. ○도 ×도 아닌 항목이 어디인지가 다음 수정 우선순위 후보. `부분적` 옆 (이유)는 반드시 적게 한다.

### critical 모두 ○ 여부

이게 No이면 정확도 100%여도 실패. 본문의 가장 중요한 부분이 전달 안 된 것.

### 불명확점

**가장 중요한 정성 데이터.** 다음 반복의 수정 대상은 이 목록에서 1개를 고른다. "불명확점 없음"이 연속 2회 나오면 수렴 신호.

### 재량 보완

서브에이전트가 자기 판단으로 채운 곳. **요건 달성 ○여도 재량 보완이 많으면 본문이 충분치 않다는 신호.** 글쓴이가 "보통 이렇게 하잖아?"라고 가정한 부분이 노출된다.

### 재시도

같은 판단을 다시 한 횟수. 본문이 모순적이거나 결정 트리가 명확치 않을 때 증가. 0이 정상.

### 메타 (usage)

정량 메트릭. tool_uses가 시나리오 간 큰 격차를 보이면 구조적 결함 신호.

## 보고 구조 미준수 시 대처

서브에이전트가 위 구조를 따르지 않고 자유 형식으로 응답하는 경우:

1. **dispatch 프롬프트에 "응답은 반드시 위 보고 구조를 따라 주십시오. 다른 형식으로 응답하면 평가가 무효화됩니다"를 추가**
2. 두 번째 시도에도 따르지 않으면, Claude가 자유 형식 응답에서 정성·정량 데이터를 추출하여 구조에 맞춰 재작성
3. 추출 시 추측이 들어가는 부분은 "추출자 보충"으로 명시

## 다중 시나리오 종합 보고

3개 시나리오를 모두 처리한 후의 종합 보고:

```markdown
## 반복 N 종합 보고서

### 시나리오 A (median - <라벨>)
정확도: <%>
critical 모두 ○: Yes/No
주요 불명확점 수: <개>
tool_uses: <M>

### 시나리오 B (edge - <라벨>)
정확도: <%>
...

### 시나리오 C (hold-out - <라벨>) — 3회차/종료 직전에만
정확도: <%>
직전 평균 대비 변동: <±N pt>
과적합 여부: <-15pt 미만 → 과적합 없음>

### 다음 반복 수정 대상
<불명확점 풀에서 1개 선정>: <이유>

### 임곗값 사전 언어화
<선정한 수정이 어느 평가 축의 어느 임곗값 문언을 충족하는가>
```

## 사용자에게 보여줄 때

종합 보고서 전체를 보여주지 말고, **요약 표 + 다음 수정 대상 + 사용자 결정 요청** 순으로 정리:

```markdown
이번 반복 결과 요약:

| 시나리오 | 정확도 | critical | tool_uses |
|---|---|---|---|
| A median | 100% | ✓ | 3 |
| B edge | 85% | ✓ | 8 |
| C hold-out | — | — | — |

수렴 후보 불명확점 (3개):
1. <항목>
2. <항목>
3. <항목>

다음 수정으로 1번을 선택하면 어느 임곗값을 충족할지
사전 언어화한 결과: <언어화 내용>

이대로 1번을 수정 대상으로 진행할까요?
```

이 형태로 사용자가 한눈에 다음 행동을 결정 가능하게 한다.
