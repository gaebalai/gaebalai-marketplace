# 전문가 아키타입 집

> 의제에 따라 최적의 전문가 패널을 구성하기 위한 참조 자료.
> 키워드 리스트가 아닌 **원리**에 근거해 동적으로 판단합니다.

## 전문가 선정의 원리

### 원리 1: 실패 감지의 원리

**"이 의제에서 실패했을 때, 누가 가장 먼저 알아차리는가?"**

그 사람의 시점을 가진 전문가가 패널에 필요합니다. 예:
- 웹사이트 레이아웃 깨짐 → 프론트엔드 엔지니어
- 사용자 이탈률 악화 → UX 리서처
- 법적 리스크 → 법무 전문가

### 원리 2: 수용자의 시점의 원리

**"이 의제의 수용자·영향을 받는 사람은 누구인가?"**

만드는 쪽뿐 아니라, 수용자(사용자, 독자, 이용자)의 시점을 포함합니다. 예:
- 기사 리뷰 → 타깃 독자 (엔지니어, 디자이너 등)
- API 설계 → API 이용자
- 사업 전략 → 엔드 유저 대변자

### 원리 3: 맹점 발견의 원리

**"이 의제에서 간과되기 쉬운 관점은 무엇인가?"**

직접 관련되지는 않지만, 중요한 시점을 가진 전문가를 포함합니다. 예:
- 이커머스 사이트 평가 → 접근성 전문가 (잊히기 쉽지만 법적 의무화되고 있음)
- 기술 아키텍처 → 비즈니스 애널리스트 (기술적으로 옳아도 사업에 맞지 않는 경우)

### 원리 4: 숨겨진 전문 지식의 원리

**"이 의제에서, 일반적으로는 소집되지 않지만 사실 중요한 지견을 가진 분야가 있는가?"**

Step 1-3에서 전문가를 선정한 후, 이 질문을 명시적으로 자문합니다.
LLM의 학습 데이터에는 "이 분야에는 이 전문성이 유효했다"는 지견이 대량으로 포함되어 있습니다.
물어보지 않으면 나오지 않지만, 물으면 끌어낼 수 있습니다.

알려진 예 (이에 한정되지 않음):
- AI 개발 → 윤리철학자 (Constitutional AI는 바로 이 성과)
- 프로덕트 설계 → 행동경제학자 (의사결정 편향, 넛지 이론)
- 조직 설계 → 생태학자 (복잡적응계, 레질리언스)
- UX 설계 → 인지심리학자 (인지 부하, 주의의 과학)
- 콘텐츠 전략 → 언어학자 (프레이밍 효과, 화용론)
- 보안 설계 → 범죄학자 (공격자의 동기 모델)

**중요: 후보가 떠오르지 않으면 무리하게 넣지 않는다.**
이 원리는 로직으로 "넣어야 한다"고 판단한 경우에만 적용합니다.
모든 의제에 숨겨진 전문가가 있는 것은 아닙니다.

### 원리 5: 대립 구조의 원리

**Devil's Advocate는 반드시 1명 포함 (전문가 슬롯과는 별도).**

비판자의 역할은 "부수는 것"이 아닌 "맹점을 발견하는 것".
가톨릭 교회가 이 직책을 폐지한 후, 시성(諡聖) 심사의 질이 저하된 역사적 사실이 있습니다.
DA는 도메인 전문가가 아닌 비판의 전문가이며, 전문가 수에는 포함하지 않습니다.

## Devil's Advocate의 초점 결정 방법

Devil's Advocate가 비판할 대상은, 의제의 성질에 따라 설정합니다:

| 의제의 성질 | Devil's Advocate의 초점 |
|---|---|
| 새로운 제안 | "왜 이것은 실패하는가" |
| 기존 시스템 | "왜 이것을 바꿔야 하는가" |
| 전략적 판단 | "이 전제가 틀렸다면 어떻게 되는가" |
| 디자인 | "가장 까다로운 사용자의 시점에서 보면 무엇이 문제인가" |
| 코드 | "6개월 후에 가장 후회할 설계 판단은 어느 것인가" |

## 전문가 수의 가이드라인

**패널 크기는 심도가 아니라, 의제가 필요로 하는 전문 영역의 수로 결정합니다.**
DA는 전문가 슬롯과는 별도로 항상 1명.

| 의제의 성질 | 직접 관련 | 수용자 | 인접 도메인 | DA | 합계 |
|---|---|---|---|---|---|
| 단일 도메인의 명확한 문제 | 2명 | 0-1명 | 0명 | 1명 | 3-4명 |
| 복수 도메인이 얽힌 문제 | 2-3명 | 1명 | 0-1명 | 1명 | 5-6명 |
| 많은 이해관계자에 영향 | 3-4명 | 1명 | 1-2명 | 1명 | 7-8명 |

- **최소**: 3명 + DA = 4명
- **최대**: 7명 + DA = 8명
- 8명을 초과하면 토큰 비용에 걸맞지 않음

### 선정의 함정 (피해야 할 패턴)

- **같은 쪽의 전문가를 2명 선정**: 편집자 + 콘텐츠 스트래티지스트 (둘 다 "만드는 쪽")
  → 대신 편집자 + 타깃 독자 (만드는 쪽 + 수용자)
- **수용자의 시점을 잊어버리기**: 전원이 "만드는 쪽", "평가하는 쪽" 전문가
  → 반드시 "사용하는 쪽", "영향을 받는 쪽"을 1명 포함
- **DA를 전문가 슬롯에 포함**: DA는 비판의 전문가이며, 도메인 전문가가 아님
  → 전문가 N명 + DA 1명으로 세기
