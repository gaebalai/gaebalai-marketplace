# 보안 룰 (전체 프로젝트 공통)

이 파일은 모든 프로젝트에서 상시 로드되는 글로벌 보안 룰입니다.

---

## 코드 생성 시의 금지 사항

- 시크릿 · API 키 · 비밀번호를 코드에 하드코딩하지 않는다 (환경 변수를 사용한다)
- SQL 쿼리에 문자열 연결을 사용하지 않는다 (플레이스홀더를 사용한다)
- 사용자 입력을 eval / exec / spawn 에 직접 전달하지 않는다
- `http://` 의 프로덕션 URL 을 설정 파일에 적지 않는다 (`https://` 를 강제)
- 응답에 스택 트레이스나 내부 경로를 포함시키지 않는다

## 보안 진단 실행 룰

- `/security-scan` 은 스테이징 환경에 대해서만 실행한다
- 프로덕션 URL (`prod`, `production`, `app.` 을 포함하는 URL) 에 대한 진단은 설정 파일에서 명시적으로 허가하지 않는 한 거부한다
- Critical 이상의 발견은 즉시 보고하고, 사람의 승인 없이 자동 수정하지 않는다
- 진단 결과에 개인 정보 · 인증 토큰의 실값이 포함될 경우에는 마스킹한 후에 기록한다

## 의존성 관리

- `npm audit` / `yarn audit` 에서 High 이상의 취약성이 검출되면 PR 을 중단한다
- 기지(旣知)의 취약한 버전 (`package-lock.json` 의 `resolved` URL) 을 직접 사용하지 않는다

## 로그 · 모니터링

- 인증 실패 · 권한 오류는 반드시 로그에 기록한다
- 로그에 평문 비밀번호 · 토큰을 포함시키지 않는다
